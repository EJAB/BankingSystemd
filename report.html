Assignment Requirements:

Customers must have the ability to log in and view their account details. Because of this, a registration process has been included to enable anyone to register with their respective details.

Customers should be able to transfer money between separate accounts and view past transactions.

Customers should be able to pay off bills

View account details and registration and account details

The home page is used as an introduction for anonymous users. Existing users have the option to login in order to access the entire application. If non-existing users wish to register, this has been provided within the login page. Once logged in, the system greets the user appropriately and the account details may be viewed using the sidebar above the logo.

If the user is logged in, he/she must first log out to view the report again.

The user has the ability to log out and then log in as a different user, which invalidates the old session and creates a new session.

Transfer money between accounts

Users have the ability to transfer money between other saved users. However, the user must input the fields correctly as they are case sensitive.

If the user tries to transfer more money than is available on his/her account, the transaction will be cancelled and logged as INVALID.

Past transactions

Users have the ability to view their past transactions, which includes the transfer amount, resulting balance, date and time of transaction and the recipient of the transaction, whether it be a person or a bill.

Pay off bills

Users have the ability to pay off bills, whether it be electricity, mortgage, phone bill, insurance, or any other outstanding loans.

SQL Statements

To create the ejab_users table:

CREATE TABLE EJAB_USERS( ID INTEGER, SURNAME VARCHAR(30) NOT NULL, FIRSTNAME VARCHAR(30) NOT NULL, USERNAME VARCHAR(30) NOT NULL, PASSWORD VARCHAR(30) NOT NULL, AMOUNT INTEGER DEFAULT(0), EMAIL VARCHAR(60) NOT NULL, PHONE VARCHAR(25), PRIMARY KEY (ID))

To create the ejab_senders table:

CREATE TABLE EJAB_SENDERS( TRANSACTIONID INTEGER, SENDERID INTEGER, SENDERFIRST VARCHAR(30), SENDERLAST VARCHAR(30), TRANSFER INTEGER, BALANCE INTEGER, TRANSACTIONTIME DATETIME, TOWARDS VARCHAR(100), PRIMARY KEY(TRANSACTIONID), CONSTRAINT CONST123 FOREIGN KEY(SENDERID) REFERENCES EJAB_USERS )

To create the ejab_receivers table:

CREATE TABLE EJAB_SENDERS( TRANSACTIONID INTEGER, RECEIVERID INTEGER, RECEIVERFIRST VARCHAR(30), RECEIVERLAST VARCHAR(30), TRANSFER INTEGER, BALANCE INTEGER, TRANSACTIONTIME DATETIME, PRIMARY KEY(TRANSACTIONID), CONSTRAINT CONST1234 FOREIGN KEY(RECEIVERID) REFERENCES EJAB_USERS )
